An analysis of machine learning using NLP

Topic modeling on ~45,000 machine learning research paper abstracts. Project by Steven Rouk.

Project overview

It's no overstatement to say that machine learning—and data science more broadly—is revolutionizing our society. The purpose of this project was to use unsupervised learning techniques (such as topic modeling, NMF, and others) to analyze the evolution of the field of machine learning over the last 20 years, including an inquiry into various subfields of machine learning and how those have risen and fallen with time.

In the menu of this website, you'll find the primary sections of analysis:

  1. How Topics Are Defined—In addition to an explanation of what a "topic" is (hint: one way to think about it is as a subfield of machine learning), the first tab shows the results of topic modeling on the arXiv.org machine learning research paper descriptions. You can see which words are associated with which topics, and the title I decided to give each topic.
  2. Change Over Time—The graphs on this page show how prevalent each topic has been over the last 20 years. One of the main results from this page is the rise of neural networks and deep learning over the last 5 years.
  3. Analysis by Year—On this page, you can conduct your own analysis by looking at varying numbers of topics for different years. I think it's interesting to see which topics existed in the early 2000s (e.g. AIXI) that don't show up in topics for modern years.
  4. All Papers—If you want to browse all of the machine learning research papers from arXiv.org, this page lets you do that.
  5. Random ML Paper—Or, if you'd prefer to randomly sample from the repository, you can jump to a random paper. For practitioners, this is an excellent way of expanding your horizons.
  6. Search—Finally, I've provided two methods of search functionality to help you find papers.

But before we dive any deeper into the details and results of the project, what is machine learning?

What is machine learning?

According to Wikipedia, "Machine learning (ML) is the scientific study of algorithms and statistical models that computer systems use to perform a specific task without using explicit instructions, relying on patterns and inference instead. It is seen as a subset of artificial intelligence."

But perhaps a better way to start this discussion is to talk about what machine learning has done in recent years.

A brief history

Let's walk through a very short history of some defining moments in machine learning from recent(ish) years. (For a bit of context, a few non-machine learning accomplishments have been listed in italicized parentheticals.)

  • (The first web browser was made available to the public in 1991.)
  • Deep Blue (by IBM) beat Garry Kasparov in chess in 1997.
  • (Google was founded in 1998.)
  • (The first iPhone was released in 2007.)
  • Watson (once again by IBM) beat Jeopardy champions Brad Rutter and Ken Jennings in 2011.
  • AlphaGo (by Google/DeepMind) beat a 9-dan professional Go player in 2016.
  • The first self-driving car completed the first DARPA Grand Challenge in 2005. By the end of 2016, Google's fleet of self-driving cars had completed over 2,000,000 autonomous miles.
  • In 2018, an NLP AI by Alibaba outperformed Stanford students on a reading and comprehension test, and Google releases "Duplex", an AI assistant with an incredibly human-like voice that can make reservations for you.

Machine learning in your life

Although self-driving cars and other futuristic technologies have been in the news much more over the last few years, we don't interact with any of those (yet). However, there are countless examples of ways we do interact with machine learning on a daily basis.

For example:

  • Every time you deposit a check, there is character recognition technology at work reading the check amount and other numbers and letters on the check. That's machine learning at work.
  • Whenever you use Google to search for something, there's a whole suite of machine learning algorithms at work trying to make sure you get the most useful results for you, independent of what other people might find helpful.
  • Digital advertisers are some of the biggest users of machine learning, using your personal data and browsing history to put relevant ads in front of you all over the web.
  • Anytime you use media services like Netflix or Spotify, or e-commerce platforms like Amazon, machine learning is being used to show you recommendations that they think you'll like.
  • Increasingly, machine learning might be used at some point in processing your insurance claims or in diagnosing medical conditions.
  • And finally, if you use Google Translate (machine learning assisted translation), Snapchat (computer vision), Facebook (image recognition), PayPal (fraud detection), or countless other online services, you're using machine learning.
  • (And we haven't even discussed at all how the government is using machine learning.)

A meteoric rise

Machine learning has exploded in recent years, in large part due to breakthroughs made possible by modern computing power, and also because of success in commercial applications of machine learning tools and approaches. This success and interest can be seen in the exponential growth of machine learning papers published on the open paper repository site arXiv.org, which was the datasource for the Evolution of Machine Learning project.

We've also seen a large increase in the public interest in machine learning, as the Google Trends data shows.

Data & Research Question

There are 1.6 million research papers hosted on arXiv.org, around 50,000 of which are categorized as machine learning papers. arXiv generously makes the paper metadata freely available through their open API, which means that it was fairly easy to pull paper descriptions, authors, etc. for all of the machine learning papers (and indeed, all of the papers).

The purpose of this project was to see if I could use natural language processing techniques on the machine learning paper descriptions to:

  1. Identify subfields of machine learning; and,
  2. See how those subfields of machine learning have evolved over time.

Method

Although there are many techniques that can be used to analyze text data, and a subset of those techniques that can be used to cluster text data into various groups, I settled on non-negative matrix factorization (NMF) as my technique of choice, paired with featurizing the data into a term-frequency document-inverse-frequency (tf-idf) matrix.

After experimenting with many different preprocessing steps, I decided to have a very simple process of removing punctuation, removing stopwords using the standard nltk corpus, and using unigrams (bag of words) to create a tf-idf matrix.

Number of Topics

For an introduction to topic modeling using NMF, see here: (insert link).

After trying many different numbers of topics, I decided to use 10 topics for the primary analysis because it gave detailed results while still being a reasonable number of manually sift through.

I thought that 20 topics also yielding very interesting results, with the added benefit that there were additional clearly-defined subfields of machine learning that didn't come out of the 10-topic analysis. To see an approximation of these topics for the full corpus, you can select 20 topics and the year 2019 or 2018 on the Analysis by Year page. (In the future I may add the functionality to look at varying numbers of topics for the full corpus.)
